## 📘 Introduction

Le **Portail DNS** est un projet académique développé à l'École Nationale Supérieure Polytechnique de Yaoundé (ENSPY) en **Mai 2024**. Ce projet vise à créer un système de gestion DNS complet avec des fonctionnalités avancées de configuration et d'administration.

---

## 🎯 Objectifs du Projet

- **Configuration DNS complète** : Mise en place d'un serveur DNS local avec zones directes et inverses
- **Gestion dynamique** : Ajout automatique d'enregistrements DNS A et PTR
- **Interface d'administration** : Interface web pour la gestion des zones DNS
- **Sécurité** : Implémentation de mesures de sécurité pour protéger le serveur DNS

---

## ⚙️ Technologies Utilisées

### Backend
- **BIND (Berkeley Internet Name Domain)** : Serveur DNS principal
- **Python** : Scripts d'automatisation et API
- **Shell Scripting** : Scripts de configuration et maintenance

### Frontend
- **HTML/CSS** : Interface utilisateur
- **JavaScript** : Interactions dynamiques
- **Bootstrap** : Framework CSS pour le design responsive

### Infrastructure
- **Linux** : Système d'exploitation serveur
- **Docker** : Conteneurisation (optionnel)
- **Git** : Gestion de version

---

## 🚀 Fonctionnalités Principales

### 1. Configuration DNS Avancée
- **Zones directes** : Résolution de noms vers adresses IP
- **Zones inverses** : Résolution d'adresses IP vers noms
- **Enregistrements multiples** : Support A, AAAA, CNAME, MX, TXT, etc.

### 2. Gestion Dynamique
- **Ajout automatique** d'enregistrements DNS A et PTR
- **Mise à jour en temps réel** des zones DNS
- **Validation** des entrées utilisateur

### 3. Interface d'Administration
- **Dashboard** de gestion des zones DNS
- **Formulaires** d'ajout/modification d'enregistrements
- **Visualisation** de la structure DNS
- **Logs** et monitoring

### 4. Sécurité
- **Contrôle d'accès** basé sur les rôles
- **Validation** des données d'entrée
- **Audit trail** des modifications
- **Protection** contre les attaques DNS
